# 수익화 설계 — 서울 결정고시 모니터

## 개요

서울시 도시계획 결정고시 모니터 서비스의 수익화 전략. 블로그 자동/반자동 발행과 광고 수익을 중심으로, 최소 운영 부담(하루 30분 이내)으로 용돈 수준(월 10~50만원)의 수익을 목표로 한다.

### 전제

- 타겟: 일반 시민 (내 동네 재개발/도시계획 소식이 궁금한 사람)
- 운영 시간: 하루 30분 이내
- 블로그와 대시보드 사이트는 독립적 수익 채널
- 향후 인천·경기 등 타 도시 확장 예정

---

## 1단계: 자동 수익 파이프라인

### 1-1. 워드프레스 자동 발행 + 애드센스

**목적:** 크롤링된 고시문을 워드프레스에 자동 발행하고, 구글 애드센스로 광고 수익을 얻는다.

**구성요소:**

- WordPress 사이트 셋업 (WordPress.com 또는 셀프호스팅)
- WordPress REST API 연동 (Application Password 인증)
- 포스팅 구조:
  - 고시 기본정보 (번호, 일자, 기관, 위치, 중심지 태그)
  - 고시 내용 (본문 텍스트)
  - 원문 이미지 (PDF → PNG 변환, 페이지별 삽입)
  - 관련 링크 (상세 페이지, 원문 다운로드)
- GitHub Actions 파이프라인: 수집 → 신규분 감지 → PDF 다운 → 이미지 변환 → WP 발행
- HWP 처리: LibreOffice → PDF 변환 후 처리 (전체의 ~1.6%)
- 구글 애드센스: 워드프레스 테마/플러그인으로 광고 삽입

**자동화 수준:** 완전 자동. 설정 후 손 댈 필요 없음.

**기술 스택:**

- `pdf2image` (poppler): PDF → PNG 변환
- `requests`: WordPress REST API 호출
- GitHub Actions: 기존 `fetch-daily.yml` 확장
- WordPress 플러그인: Ad Inserter 또는 Site Kit (애드센스 연동)

### 1-2. 네이버 블로그 반자동 발행 + 애드포스트

**목적:** 네이버 검색 유입을 확보하고, 네이버 애드포스트로 수익을 얻는다.

**현재 상태:** admin 페이지(`/admin`)에서 "블로그 쓰기" 버튼 → 클립보드 복사 → 네이버 블로그 글쓰기 페이지 오픈 → 붙여넣기. 이미 구현 완료.

**추가 작업:**

- 네이버 애드포스트 신청 (일 방문자 조건 충족 시)
- 포스팅 제목 SEO 최적화 (지역명+키워드 패턴)
  - 예: "강남구 재건축 결정고시 (2026.03.20) — 삼성동 일대 용적률 변경"
- 블로그 포스팅 하단에 대시보드 사이트 링크 삽입 (선택적 유입)

**자동화 수준:** 반자동. 하루 10~15분, 신규 고시문 확인 후 "블로그 쓰기" 클릭.

**완전 자동화 불가 사유:** 네이버 API 미제공, Selenium은 계정 정지 리스크.

### 1-3. 대시보드 사이트 애드센스

**목적:** 대시보드 사이트 자체에 구글 애드센스를 삽입하여 방문자로부터 광고 수익을 얻는다.

**광고 위치:**

- 메인 대시보드(`/`): 고시문 목록 하단 배너
- 서울플랜 페이지(`/plan`): 섹션 사이 네이티브 광고
- 강북전성시대 페이지(`/gangbuk`): 섹션 사이 네이티브 광고
- 모바일: 하단 고정 배너

**구현 방식:**

- `AdBanner` 공통 컴포넌트 생성
- 구글 애드센스 스크립트를 `layout.tsx`에 삽입
- 광고 슬롯을 각 페이지에 배치
- `/admin` 페이지에는 광고 미삽입

**자동화 수준:** 완전 자동. 코드 한 번 삽입하면 끝.

**주의:** 애드센스 승인 조건 — 충분한 콘텐츠, 개인정보처리방침, 약관 페이지 필요.

## 2단계: 콘텐츠 차별화 (1~2개월 후)

### 2-1. 주간 자동 분석 리포트

**목적:** 크롤링 데이터를 기반으로 주간 요약 리포트를 자동 생성하여 SEO 트래픽을 확보한다.

**리포트 구성:**

- 이번 주 신규 고시문 건수
- 중심지별 분포 (어디에서 가장 많았나)
- 주요 키워드 (재개발, 용적률, 용도변경 등)
- 전주 대비 변화
- 주목할 고시문 1~2건 하이라이트

**생성 방식:**

- Python 스크립트: 주간 데이터 집계 → 마크다운/HTML 생성
- GitHub Actions: 매주 월요일 자동 실행
- 워드프레스 + 네이버 동시 발행

**자동화 수준:** 완전 자동 (설정 후).

### 2-2. SEO 최적화

**블로그 제목 패턴:**

```
[지역구] [키워드] 결정고시 ([날짜]) — [상세 위치/내용]
```

- 예: "영등포구 재건축 결정고시 (2026.03.18) — 여의도동 용적률 변경"
- 예: "노원구 도시계획 결정고시 (2026.03.15) — 상계동 정비구역 지정"

**목표:** 네이버 검색에서 "OO구 결정고시", "OO동 재개발" 등의 키워드로 상위 노출. 경쟁 키워드가 거의 없는 니치 분야이므로 상위 노출 가능성 높음.

### 2-3. LLM 자동 요약/해설

**목적:** 고시문마다 일반인이 이해할 수 있는 요약과 해설을 자동 생성하여, 애드센스 심사 시 "고유 콘텐츠"로 인정받고 SEO 품질을 높인다.

**생성 내용 (JSON 응답):**

- `summary`: 2~3줄 요약 (이 고시가 뭔지)
- `impact`: 일반인 해설 1~2줄 (주변 부동산/생활에 미칠 영향)
- `policy_context`: 관련 정책과의 연결고리 (관련 없으면 null)
- `keywords`: 핵심 키워드 3~7개 (태그 자동 부여와 연계)

**워크플로우:**

```
publish_daily.py
  │
  ├─ 고시문 로드
  │
  ├─ 위치/키워드로 관련 정책 판별
  │   ├─ 관련 정책 있음 → policy_reference.md를 컨텍스트로 포함
  │   └─ 관련 정책 없음 → 원문만으로 인사이트 추출
  │
  ├─ Claude API 호출 (prompts/insight_prompt.md 기반)
  │   └─ 시스템 프롬프트 + 고시문 + 정책 레퍼런스(해당 시)
  │
  ├─ JSON 응답 파싱
  │   ├─ summary + impact + policy_context → wp_blog_template "한눈에 보기" 섹션
  │   └─ keywords → WP 태그 API 자동 부여
  │
  └─ WP 발행
```

**정책 레퍼런스 파일:**

- `prompts/policy_reference.md` — 3대 정책(2040 서울플랜, 강북전성시대 2.0, 서남권 대개조 2.0) 핵심 데이터
- `prompts/insight_prompt.md` — LLM 프롬프트 템플릿 (시스템 + 유저 프롬프트)
- 고시문 위치가 정책 관련 지역에 해당하면 레퍼런스 포함, 아니면 원문만으로 요약

**구현 파일:**

- `insight_generator.py` — Claude API 호출 + 응답 파싱 모듈
- `publish_daily.py` — 발행 전 인사이트 생성 호출 추가
- `wp_blog_template.py` — "한눈에 보기" 섹션 삽입 로직 추가

**환경변수:** GitHub Secrets에 `ANTHROPIC_API_KEY` 추가

**비용:** 하루 1~5건 × 건당 ~10원 = 월 1,000원 미만

**자동화 수준:** 완전 자동.

### 2-4. 키워드 자동 태깅 (2-3과 통합 가능)

**목적:** 고시문 제목/본문에서 핵심 키워드를 자동 추출하여 WP 태그로 부여. SEO 강화 + 키워드별 모아보기.

**추출 키워드:**

- 재건축, 재개발, 용적률, 용도변경, 지구단위계획, 정비구역, 실시계획, 도시계획시설 등
- 고시문에 자주 등장하는 도시계획 용어 사전 기반 매칭

**구현:** Python에서 키워드 사전 매칭 → WP 태그 API로 자동 부여

**자동화 수준:** 완전 자동.

### 2-5. 지역 핫스팟 분석

**목적:** 최근 3개월 고시 집중 지역 TOP5를 자동 분석하여 대시보드에 "지금 뜨는 지역" 섹션 추가.

**표시 내용:**

- 구별 고시 건수 랭킹
- 전월 대비 증감
- 핫스팟 지역 하이라이트 카드

**구현:** 대시보드 컴포넌트 추가 (기존 데이터 기반 집계)

### 2-6. 인스타그램 자동 발행

**목적:** 고시문을 비주얼 콘텐츠로 가공하여 인스타그램에 자동 발행. SNS 유입 채널 확보.

**콘텐츠 구성 (캐러셀 2장):**

- 1장: 썸네일 — 등급별 컬러 + 지역명 + 제목 + 날짜 (기존 thumbnail.py 활용)
- 2장: 요약 카드 — LLM 요약 + 영향 분석 텍스트를 이미지로 생성 (Pillow)

**캡션:**

- LLM 요약 2~3줄
- 영향 분석 1줄
- 해시태그 자동 생성: #서울결정고시 #OO구재건축 #도시계획 #용적률 등
- 프로필 링크로 대시보드 유도

**기술 스택:**

- Instagram Graph API (무료)
- Pillow: 요약 카드 이미지 생성 (thumbnail.py 확장)
- 한글 폰트: fonts-noto-cjk (GitHub Actions에 이미 설치)

**사전 준비 (수동):**

- 인스타 비즈니스/크리에이터 계정 전환
- Facebook 페이지 생성 + 인스타 계정 연동
- Meta Developer 앱 생성 + 토큰 발급
- GitHub Secrets: `INSTAGRAM_ACCESS_TOKEN`, `INSTAGRAM_ACCOUNT_ID`

**워크플로우:**

```
publish_daily.py
  └─ WP 발행 후
      ├─ 썸네일 (기존) + 요약 카드 이미지 생성
      ├─ 이미지 → Instagram Graph API 업로드
      └─ 캡션 (LLM 요약 + 해시태그) 자동 생성
```

**주의:** 액세스 토큰 60일 갱신 필요 (장기 토큰 자동 갱신 로직 구현 가능)

**비용:** 무료 (API 비용 없음)

**자동화 수준:** 완전 자동 (Meta 앱 심사 + 토큰 발급 후).

### 2-7. 알림 기능

**목적:** 관심 지역/키워드 고시 감지 시 실시간 알림.

**채널:**

- 텔레그램 봇 (무료, 개인용 적합)
- 슬랙 webhook (팀용)
- 이메일 (선택)

**구현:** GitHub Actions에서 신규 고시 감지 시 조건 매칭 → 알림 발송

**자동화 수준:** 완전 자동 (초기 관심 키워드/지역 설정 후).

## 3단계: 범주 확장 + 도시 확장 (트래픽 안정화 후)

### 3-0. 서울도시공간포털 범주 확장

**목적:** 현재 결정고시만 수집 중인 크롤러를 다른 범주로 확장하여 콘텐츠 양 증가 + 대시보드/블로그 커버리지 확대.

**API 조사 결과 (2026.03.23):**

| 범주 | API 엔드포인트 | 데이터 건수 | 상태 |
|------|---------------|-----------|------|
| 결정고시 | `ntfc/getNtfcList.json` | 43,634건 | 수집 중 |
| 지구단위계획 | `dstplan/getDstplanList.json` | 4,313건 | API 확인됨 |
| 도시계획시설 | 미확인 | - | 엔드포인트 조사 필요 |
| 용도지구 | 미확인 | - | 엔드포인트 조사 필요 |
| 용도구역 | 미확인 | - | 엔드포인트 조사 필요 |
| 정비사업구역계 | 미확인 | - | 엔드포인트 조사 필요 |
| 도시개발사업 | 미확인 | - | 엔드포인트 조사 필요 |

**구현 시 필요한 작업:**

- 크롤러 확장: 범주별 API 호출 + 데이터 파싱
- `GosiRecord`에 `category` 필드 추가
- 대시보드: 범주별 필터 탭
- WP 발행: 범주별 카테고리 매핑

**우선순위:** 지구단위계획부터 추가 (API 확인 완료), 나머지는 엔드포인트 조사 후

### 3-1. 타 도시 크롤러 추가

**대상 후보:**

- 인천시 (인천도시공간포털 또는 유사 API 조사)
- 경기도 주요 도시 (수원, 성남, 고양 등)
- 부산, 대구 등 광역시

**구현 방식:**

- 크롤러 모듈화: 도시별 설정(API URL, 파싱 규칙)만 교체하는 구조
- 데이터 구조 통일: 기존 `GosiRecord` 타입에 `city` 필드 추가
- 대시보드 재사용: 도시 선택 필터 추가, 나머지 컴포넌트는 그대로
- 경로: `/incheon`, `/gyeonggi` 등 도시별 페이지 또는 메인에서 필터

### 3-2. 브랜드 확장

- 현재 "서울 결정고시 모니터" → "도시계획 결정고시 모니터" 등으로 전환 가능
- 도메인 확보 고려
- 도시별 블로그 카테고리/태그 분리로 SEO 극대화

## 제약 및 리스크

- **애드센스 승인:** 콘텐츠 양과 품질, 개인정보처리방침 필요. 초기에 거절될 수 있음.
- **네이버 애드포스트:** 일 방문자 수 기준이 있음 (비공개). 블로그 운영 기간도 영향.
- **트래픽:** 고시문은 니치 분야라 초기 트래픽이 적을 수 있음. SEO 최적화로 점진적 성장 필요.
- **네이버 블로그 자동화:** API 미제공으로 완전 자동화 불가. 현재 반자동 유지.
- **도시 확장 시:** 각 도시의 고시 API가 다를 수 있어 크롤러 파싱 로직 개별 개발 필요.
- **저작권:** 고시문 원문은 공공저작물(공공누리)이므로 출처 표시 시 자유 이용 가능. 다만 도시공간포털 이미지(서울플랜 페이지)는 공공누리 유형 확인 필요.
